## 0. Главная заповедь

**Коран — Слово Аллаха. Ни одна буква не должна быть искажена.**

Каждый аят, каждое огласовление (харакат), каждый знак таджвида должны побайтово совпадать с эталоном:
- **Mushaf al-Madinah** (риваят Хафс от Асима) — основной эталон.
- Источник данных: [Tanzil.net](https://tanzil.net/docs/quran_text) — текст `quran-uthmani.txt` (UTF-8, Unicode).
- Перекрёстная проверка: Quran.com API v4 (corpus).
- Перед каждым релизом — sha256-проверка файлов с текстом Корана.

**Если есть сомнение в тексте — не угадывать. Открыть физический Мусхаф или Tanzil.net.**

## 4. Правила написания кода

### 4.1. Арабский текст и RTL
- Все арабские строки — в отдельных i18n-ключах, **никогда не хардкодить арабский в коде**.
- RTL-поддержка: использовать `I18nManager.forceRTL()` только для полностью арабского UI.
- Для смешанного контента (арабский + латиница) — `writingDirection` и `textAlign` на уровне компонента.
- Шрифт арабского Корана — только KFGQPC Uthman Taha Naskh (или Uthmanic Hafs). Никаких системных шрифтов.

### 4.2. Коранический контент
- Текст аята **всегда** рендерится из проверенного JSON-файла, никогда не из строки в коде.
- SHA-256 хеш JSON-файла с текстом Корана хранится в `assets/quran-checksum.json` и проверяется при старте.
- Номера аятов и сур — 1-индексированные (как в Коране).

### 4.3. Оффлайн-первый
- Любой экран должен работать без сети. Если нужна сеть — явный индикатор и фоллбэк.
- Текст Корана, базовый перевод (Кулиев), аудио одного чтеца (Мишари) — всегда в bundle или скачаны при первом запуске.
- Кеш — через WatermelonDB / SQLite, не `AsyncStorage` для больших данных.

### 4.4. Конфиденциальность
- Никаких external SDK без ревью (Firebase Analytics, Meta SDK — запрещены).
- Sentry — только при явном согласии пользователя, self-hosted при возможности.
- Заметки и журнал намерений — только локально, без передачи на сервер в открытом виде.

## 5. Визуальные правила

- **Никаких изображений живых существ** (людей, животных) нигде в приложении.
- Иллюстрации — только геометрия, архитектура, книги, звёзды, луны, орнаменты.
- Каллиграфические SVG — только из проверенных источников. Перед добавлением — убедиться, что аят передан верно.
- Анимации — мягкие, без «игровых» эффектов (фейерверки, монеты).
- Цветовая система — строго из design-tokens (см. раздел 8.1 спецификации).

## 6. Что проверять перед каждым коммитом

- [ ] Текст Корана не изменён вручную
- [ ] Нет изображений людей или животных
- [ ] Нет хардкоженного арабского текста в коде (только через i18n / данные)
- [ ] Нет внешних аналитических SDK без согласия
- [ ] RTL работает корректно для арабского UI
- [ ] Оффлайн-режим не сломан
- [ ] Шрифт арабского — Uthmanic, не системный
